Linux Ansible

Ansible 原理

自动化运维工具:

  1. puppet 需要master和agent
  2. stltstack 需要master和agent
  3. ansible 不需要部署agent

上线一个业务，部署一个网络集群

  1. 服务器

  2. 上架服务器

  3. 配置网络

  4. 安装操作系统

  5. 系统初始化（配置网卡，主机名，装包，ssh）

  6. 安装中间件（httpd，nginx）

  7. 发布

    *ansible基于ssh python来进行管理** 瓶颈也在ssh,ssh天生不支持高并发，性能会下降

    *模块化管理** file shell command

    aster: 控制端 不依赖任何进程 ansible提供命令行

    gentless: 主机清单(hosts inventory) 在master上，被控节点上，不需要安装额外软件包以及额外进程

Ansible原理

  1. ansible如何去多台机器上执行命令

    通过主机清单进行管理（只有在主机清单里面的主机，才可以使用ansible来进行管理）

    使用ansible指令，将我们的命令翻译成Python脚本，xxxx.py

    通过ssh将xxxx.py发送到远程主机，然后再远程主机执行该脚本

  2. ansible如何识别被控端

    通过主机清单以及ssh识别

Ansible特性

  • Agentless：不需要在被控节点安装客户端，有sshd即可
  • Serverless：在服务端不需要启动任何服务，只要执行命令即可
  • Modules in any language：基于模块工作，可用任何语言开发模块
  • YAML, not code：使用yaml语言定制playbook
  • SSH by default：默认使用ssh控制各节点
  • Strong multi-tier solution：可实现多级控制（可以摆脱ssh连接数的限制）

Ansible基本组件

  • 核心：ansible
  • 核心模块（Core Modules）：ansible自带
  • 扩展模块（Custom Modules）：核心模块不足以完成某种功能时，添加扩展模块
  • 插件（Plugins）：完成模块功能的补充
  • 剧本（Playbooks）：把需要完成的多个任务定义在剧本中
  • 连接插件（Connectior Plugins）：ansible基于连接插件连接到各个主机上，ansible支持ssh以外的连接方式，所以需要连接插件
  • 主机清单（Host Inventory）：ansible在管理多台主机时，可以有选择性地对其中一部分进行操作

Ansible工作机制

​ Ansible在管理节点将Ansible模块通过SSH协议（或Kerberos,LDAP）推送到被管理端执行，执行完后制动删除，可以使用版本控制系统（git/svn）来管理自定义模块以及playbooks

Ansible安装

# yum -y install ansible
# ansible --version

基于yum安装时，需要配置epel源
