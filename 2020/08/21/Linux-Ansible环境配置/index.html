Linux-Ansible环境配置

2020-08-21 Linux 755 words
Linux Ansible

Ansible环境配置

至少需要三台机器，越多越好

网络规划：如果最小化安装，在安装操作系统的时候，就最好把网络和主机名设置好，要设置开机自动启动

配置controller

配置本地yum源（使用两个镜像文件，一个系统iso，一个ansible的）

创建挂载点

[root@ansible ~]# mkdir /iso

[root@ansible ~]# mkdir /iso2

写入自动挂载

[root@ansible ~]# vim /etc/fstab

/dev/sr0                /iso                    iso9660 defaults,loop   0 0
/dev/sr1                /iso2                   iso9660 defaults,loop   0 0

配置本地yum源

[root@ansible ~]# vim /etc/yum.repos.d/dvd.repo

[appstream]
name=app
baseurl=file:///iso/AppStream
gpgcheck=0
enabled=1

[base]
name=base
baseurl=file:///iso/BaseOS
gpgcheck=0
enabled=1

下载并配置ftp服务

[root@ansible ~]# yum install vsftp -y

[root@ansible ~]# systemctl enable –now vsftpd

[root@ansible ~]# vim /etc/vsftpd/vsftpd.conf

anonymous_enable=YES
anon_root=/iso

通过ftp共享yum源给其他机器使用

关闭防火墙

[root@ansible ~]# systemctl stop firewalld
[root@ansible ~]# systemctl disable firewalld
Removed /etc/systemd/system/multi-user.target.wants/firewalld.service.
Removed /etc/systemd/system/dbus-org.fedoraproject.FirewallD1.service.

配置SELinux

[root@ansible ~]# vim /etc/selinux/config

SELINUX=disabled

重启服务

[root@ansible ~]# systemctl restart vsftpd

测试

共享yum源，后续也可以使用ansible完成

拷贝

[root@ansible ~]# cp /etc/yum.repos.d/dvd.repo ftp.repo

编写

[root@ansible ~]# vim ftp.repo

[appstream]
name=app
baseurl=ftp://192.168.1.200/AppStream
gpgcheck=0
enabled=1

[base]
name=base
baseurl=ftp://192.168.1.200/BaseOS
gpgcheck=0
enabled=1

将yum拷贝到各个节点

[root@ansible ~]# scp ftp.repo root@192.168.1.201:/etc/yum.repos.d

在controller配置ansible的yum源

要把ansible的iso文件挂载起来

[ansible]
name=ansible
baseurl=file:///iso2/ansible-2.8-for-rhel-8-x86_64-rpms
gpgcheck=0
enable=1

安装ansible

[root@ansible ~]# yum install ansible -y

Complete!

初始化ansible

  1. 创建一个普通用户，使用普通用户来管理ansible

    把所有用户放到一个组里

    [root@ansible ~]# cd /etc/ansible/
    [root@ansible ansible]# ls
    ansible.cfg hosts roles
    [root@ansible ansible]# vim hosts

    [node]
    192.168.1.201
    192.168.1.202
    192.168.1.203

    关闭 host_key_checking

    [root@ansible ansible]# vim ansible.cfg

    host_key_checking = False

    统一创建用户

    [root@ansible ansible]# ansible node -m shell -a “useradd ansible” -k
    SSH password:

    [root@ansible ansible]# ansible node -m shell -a “echo 000000 | passwd –stdin ansible” -k
    SSH password:

  2. 进行sudo提权

    [root@ansible ansible]# cd /etc/sudoers.d/
    [root@ansible sudoers.d]# touch ansible
    [root@ansible sudoers.d]# vim ansible

    ansible ALL=(root)      NOPASSWD: ALL

    [root@ansible sudoers.d]# ansible node -m shell -a “echo ‘ansible ALL=(root) NOPASSWD:ALL’ > /etc/sudoers.d/ansible” -u root -k
    SSH password:

  3. 修改ansible配置文件，进行自动提权，及修改默认用户

    [root@ansible ansible]# vim ansible.cfg

    remote_user = ansible

    [privilege_escalation]
    become=True
    become_method=sudo
    become_user=root
    become_ask_pass=False

做ssh免密登录

  1. 本地域名解析

    [root@ansible ansible]# vim /etc/hosts

    192.168.1.200   ansible
    192.168.1.201   node1
    192.168.1.202   node2
    192.168.1.203   node3

  2. 生成ssh密钥

    全部回车

    yes

    [root@ansible ssh]# ansible node -m shell -a “systemctl is-active httpd”
